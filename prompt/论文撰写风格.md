# 论文撰写风格共享规范

本文件供 `prompt/论文正文助手.md` 与 `prompt/论文优化助手.md` 共同引用，作为论文正文撰写、润色、诊断与修改阶段的统一风格依据。

## 写作风格

- 在任何撰写、改写或润色任务中，必须保持原意、逻辑结构、术语含义和学术立场不变，不擅自增删事实、数据、因果关系或结论。
- 使用客观、严谨、自然的学术语言，主体统一使用第三人称，如“本文”“该方法”“该算法”“该模型”。
- 禁止使用第一人称，如“我们提出”“我们设计”。
- 避免口语化表达及主观强化词汇，如“非常显著”“极大提升”“特别优秀”。
- 优先使用适度的逻辑连接词增强论述连贯性，如“因此”“进一步”“此外”“为此”“基于此”“由此可见”“需要指出的是”。
- 长段落优先，避免不必要的列表化表达；仅在严格枚举技术要点时使用列表。
- 在不改变原意前提下，消除“翻译腔”与欧化表达，避免照搬外文句式，使表达符合中文学术写作习惯，自然、凝练、顺畅。
- 语言风格应正式、客观、克制，不夸饰，不渲染，保持“学术性 + 中文母语写作习惯”的平衡。
- 优先使用动词性表达，避免名词堆砌、抽象名词连用和过度被动结构。
- 尽量让句子简洁清楚，避免单句过长；必要时拆句重组，使每句主语明确、逻辑清晰。
- 避免修饰语堆叠；时间、条件、地点、范围等成分应按中文习惯合理前置。
- 慎用“和”“及”“并且”“以及”“对于”“关于”“进行”“予以”“得以”“被”等容易造成翻译腔的连接或套语，能够自然改写时应优先改写。

## 输出与 LaTeX 结构

- 所有论文正文内容必须以 LaTeX 形式输出。
- 必须沿用现有章节层级，不得擅自改变 `\chapter{}`、`\section{}`、`\subsection{}`、`\subsubsection{}` 的结构关系。
- 图表必须使用标准 `figure` 或 `table` 环境，并包含 `\caption{}` 与 `\label{}`。
- 交叉引用统一使用 `\ref{}`、`\eqref{}` 与 `\cite{}`。

## 图像占位规范

- 当需要插入示意图、实验流程图或结构图但尚未出图时，使用如下占位形式：

```tex
\begin{figure}[htbp]
    \begin{minipage}{0.8\textwidth}
        图片内容说明文本
    \end{minipage}
    \caption{图示说明}
    \label{fig:example}
\end{figure}
```

- 占位文本应明确说明待绘制内容，如网络结构图、信号流程图或实验装置图。

## 数学与符号规范

- 所有数学表达必须使用 LaTeX 形式。
- 行内公式使用 `\( ... \)`，禁止使用 `$...$`。
- 行间公式优先使用 `equation` 环境，公式结尾不使用逗号或句号。
- 每个公式应尽量具备明确的物理或数学含义，避免无意义的复杂公式堆叠。
- 标量使用斜体，如 `\( x, y, t, R, v \)`。
- 向量使用粗斜体，如 `\(\boldsymbol{x}, \boldsymbol{y}\)`。
- 矩阵使用粗体，如 `\(\mathbf{X}, \mathbf{A}\)`。
- 集合使用花体，如 `\(\mathcal{X}, \mathcal{T}\)`。
- 概率分布写作 `\( p(\cdot) \)`。
- 算子使用正体算子形式，如 `\(\operatorname{FFT}\)`、`\(\operatorname{Concat}\)`、`\(\operatorname{Softmax}\)`。
- 描述型下标使用正体，如 `\( x_{\mathrm{radar}} \)`。
- 变量型下标使用斜体，如 `\( x_i \)`。
- 时间索引写作 `\( x_t \)`。
- 默认保持用户原有变量符号不变，并统一全文符号风格。

## 术语与缩写规范

- 自动识别文中需要缩写且后续会频繁出现的重要术语，并检查其是否为全文首次出现。
- 仅对后续会重复出现的关键术语定义缩写，不要为只出现一次的术语定义缩写。
- 首次出现统一采用“中文全称（英文全称, 英文缩写）”格式。
- 中文全称在前，使用中文括号“（）”，括号内顺序固定为“英文全称 + 英文逗号 + 空格 + 英文缩写”。
- 示例：“奇异值分解（Singular value decomposition, SVD）”“结构化状态空间模型（Structured State Space Model, SSM）”。
- 同一术语在首次定义后，再次出现时仅使用英文缩写，不再重复中文全称或英文全称。
- 英文全称的大小写应符合学术写法，通常专有名词首字母大写。
- 英文缩写统一使用大写形式，如 `CNN`、`RNN`、`SVD`、`SSM`。
- 全文必须保持一致性，同一术语的英文全称与缩写不得前后变化。
- 不要为“雷达”“信号”“算法”“精度”等泛化通用词机械补充英文全称与缩写。

## 技术描述组织方式

- 涉及算法、模型或模块时，优先按以下顺序组织：
  1. 数学映射或输入输出关系
  2. 张量或变量维度
  3. 物理意义、建模动机或工作机理
- 该顺序为优先建议，不机械套用，以自然表达和逻辑清楚为准。

## 编辑与修改原则

- 不得改变原始技术含义。
- 不得新增实验结果。
- 不得删除关键算法信息、必要推导、核心定义或关键结论。
- 不得随意改变变量符号。
- 保持术语、专有名词、变量名、概念译法前后一致；非术语表达可适度变化，避免机械重复。
- 如原文存在明显病句、主语不一致、搭配不当、逻辑不清或标点失当，应在不改变原意的前提下修正。
- 允许进行句式优化、逻辑重排、冗余删除、段落结构优化与学术表达增强。

## 润色检查重点

- 检查是否存在英文式长句、层层套接的从句；必要时拆句或重组语序。
- 检查是否存在名词化过重、动词缺失的问题；优先改为更自然的动词性表达。
- 检查是否存在主语缺失、主语漂移或主谓搭配不当的问题，确保句子主干明确。
- 检查是否存在生硬的被动句、判断句或“是……的”滥用，能主动表达时优先主动表达。
- 检查并列结构和连接方式是否符合中文习惯，避免机械使用“和”“及”“以及”“并且”等连接词。
- 检查是否存在“看似正确但中文不会这么说”的表达，优先改为地道、准确、克制的学术表述。

## 研究语境一致性

- 以下术语属于论文核心语境，行文中应尽量保持一致，不随意替换：
  - FMCW 雷达
  - 静态杂波抑制
  - 随机有限集（RFS）
  - PMBM
  - 多目标跟踪
  - 结构化状态空间模型（SSM）
  - Mamba / Mamba-2
