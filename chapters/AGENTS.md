# `chapters/` Agent 总入口

本文件适用于 `chapters/` 目录树内所有 `.tex` 文件（章节、摘要、附录等），并作为本目录论文任务的统一入口。

## 角色定位

- 本目录下的论文任务统一分为两个功能入口：
  - **撰写**：调用 `prompt/论文正文助手.md`
  - **优化**：调用 `prompt/论文优化助手.md`
- 两个功能共用同一套写作、数学、术语与 LaTeX 规范：
  - `prompt/论文撰写风格.md`

## 功能入口与调用约定

- 当任务是论文撰写、补写、扩写、改写、局部润色，且目标是产出可直接落入正文的内容时：
  - 使用 `prompt/论文正文助手.md`
  - 同时遵循 `prompt/论文撰写风格.md`
- 当任务是长章节检查、问题诊断、重复排查、符号统一、风格统一、逻辑矛盾检查时：
  - 使用 `prompt/论文优化助手.md`
  - 同时遵循 `prompt/论文撰写风格.md`
- 当任务同时包含“先诊断、再修改”两个阶段时：
  1. 先按 `prompt/论文优化助手.md` 进入诊断模式
  2. 在用户确认修改范围后，再按 `prompt/论文正文助手.md` 进行局部改写

## 规则来源与优先级

- 除遵循仓库根 `AGENTS.md` 外，`chapters/` 目录写作任务必须遵循本文件。
- `prompt/论文撰写风格.md` 是本目录撰写风格相关规则的唯一共享来源。
- 若规则冲突，遵循“更具体、更严格”原则：
  - 目录工作流与 WIP 约定：以本文件为主
  - 撰写输出：以 `prompt/论文正文助手.md` 为主
  - 诊断与修改流程：以 `prompt/论文优化助手.md` 为主
  - 风格、数学、术语、LaTeX 规范：以 `prompt/论文撰写风格.md` 为主

## 任务边界

- 当任务是论文撰写、润色、补写、改写时，输出应为 LaTeX 正文内容。
- 当任务是概念、算法或理论解释时，可直接进行技术说明，不强制使用论文正文格式。

## WIP 约定（进行中内容）

- **进行中内容固定写在 `chapters/aaaa.tex`。**（在`aaaa.tex`顶部注释标明引用位置）
- 当前正在写的章节文件中用 `\input{chapters/aaaa.tex}` 引用它。


## 目录级补充规则（强制）

- 不要在 `.tex` 中保留 Markdown 风格 `-` 列表；需要列表时使用 `itemize` 或 `enumerate`。
- 需要保留草稿提示时，优先使用 `%` 注释，而不是将待办内容直接混入正文。
- 图表仅引用 `figures/` 下最终图片，不在正文直接引用脚本目录临时图。
- 图题与标签顺序统一为先 `\caption{...}`，后 `\label{...}`。
- 公式、图表、章节与文献交叉引用的 label 命名应保持稳定，避免无关改名导致全文引用失效。
- 不需要编译验证语法问题

## 全文结构快速索引

- 当 agent 需要快速了解全文结构时，调用 `scripts/toc_to_markdown.py` 生成正文标题索引文件。
- 默认执行命令：
  - `.\.venv\Scripts\python.exe scripts\toc_to_markdown.py -o build\论文正文标题.md`
- 输出文件固定为：
  - `build/论文正文标题.md`
